\# Splunk Fundamentals (SOC Perspective)



Three key Splunk concepts were reviewed before ingesting logs.



\## Index



An index is where Splunk stores ingested logs.



Example indexes:



main  

windows  

linux  

firewall  

ids  



For this lab, logs were stored in the main index.



\## Source



The source identifies where the log originated.



Example:



/var/log/auth.log



This source contains Linux authentication telemetry including SSH activity.



\## Sourcetype



The sourcetype tells Splunk how to interpret and parse the log format.



For Linux authentication logs the sourcetype used was:



syslog



This enables Splunk to correctly parse fields such as:



\- host

\- process

\- sshd

\- sudo

\- authentication messages

\# Log Ingestion Configuration



Security logs were ingested using Splunk’s monitoring capability.



Configuration:



Input Type: File Monitor  

Source Path: /var/log/auth.log  

Monitoring Mode: Continuous  

Sourcetype: syslog  

Host: wazuh-siem  

Index: main  



This allowed Splunk to continuously ingest Linux authentication events in real time.

\# Initial SPL Queries



After confirming ingestion, several SPL queries were executed to validate visibility and perform basic investigation.



\## Retrieve Linux Authentication Logs



index=main source="/var/log/auth.log"



This query returns all Linux authentication telemetry.



\## Detect Failed SSH Logins



index=main source="/var/log/auth.log" "Failed password"



This query identifies SSH authentication failures which may indicate brute-force activity.



\## Detection-Style Query (Counting Failed Logins)



index=main source="/var/log/auth.log" "Failed password"  

| stats count by host



This query counts authentication failures by host and represents the basic logic used in brute-force detection.



\## SOC Investigation Query



index=main source="/var/log/auth.log"  

("Failed password" OR "Accepted password" OR "sudo")  

| table \_time host \_raw



This query provides a consolidated view of:



\- Failed SSH logins

\- Successful SSH logins

\- Privilege escalation activity via sudo



This type of query supports rapid investigation during authentication-related security incidents.
